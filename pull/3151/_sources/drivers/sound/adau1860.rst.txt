~~~

The I2C device id depends on the ADDR0 and ADDR1 pin settings and needs to be
set according to your board setup.

   :header-rows: 1

   * - ADDR1

   * - 0
     - 0
     - 0x64
   * - 0
     - 1
     - 0x65
   * - 1
     - 0
     - 0x66
   * - 1
     - 1
     - 0x67

In this example we assume ADDR0=0 and ADDR1=0.

ALSA controls can be modified easily using alsa-tools utilities. For example to
read the current state of *DMIC0 Capture Volume* control one can use:

``amixer -c ADAU1860 cget name=‘DMIC0 Capture Volume’``

To set one control value the following command can be executed:

``amixer -c ADAU1860 cset name=‘DMIC0 Capture Volume’ 191``

DAI configuration
~~~~~~~~~~~~~~~~~

The codec driver registers two DAIs:

- adau1860-sai0
- adau1860-sai1
